# Example Use Cases

## 1. Management reporting
Prepare a consistent data source for dashboards and regular management reporting.

Typical outputs:
- project overview
- work item status distribution
- overdue items

## 2. Operational transparency
Give stakeholders visibility into changes, bottlenecks, and planning shifts.

Typical outputs:
- status change history
- delayed items
- blocked work
- reassignment patterns

## 3. Baseline vs. current-state analysis
Compare a planned or reference state with the current operational state.

Typical outputs:
- date deviations
- changed work item counts
- delayed milestones
- planning drift indicators

## 4. Notification support
Prepare structured data for automated notifications to responsible users or stakeholders.

Typical triggers:
- overdue item
- status changed to blocked
- upcoming deadline
- change in assignment
- critical schedule shift

## 5. Historical analysis
Support analysis of how and why operational changes occurred over time.

Typical questions:
- when did the status change
- what changed between snapshots
- which items changed most often
- which process areas show repeated delays

## 6. Reduced manual reporting
Replace spreadsheet-based or manually compiled reporting with a repeatable data pipeline.

Typical benefits:
- fewer manual steps
- more reliable reporting
- easier auditability
- clearer source of truth
